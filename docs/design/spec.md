# Distill - 機能仕様書

## 1. システム概要

Distillは動画URLを入力するだけで、文字起こし・要約・コンテンツ変換を行うシングルページWebアプリケーション。

### システムフロー

```
[ユーザー] → URL入力 → [Next.js API Route]
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
              [URL解析]           [メタ情報取得]
              プラットフォーム判定   タイトル・チャンネル名
                    │
                    ▼
              [音声抽出]
              yt-dlp / ffmpeg
                    │
                    ▼
              [Claude API]
              音声 → 文字起こし
                    │
                    ▼
              [Claude API]
              テキスト → 要約・変換
                    │
                    ▼
              [レスポンス]
              全文・要約・ブログ・SNS
                    │
                    ▼
              [ブラウザ]
              タブ表示 + PDF保存
```

## 2. URL解析・音声抽出

### 2.1 プラットフォーム判定

| プラットフォーム | URL パターン | 抽出方法 |
|----------------|-------------|---------|
| YouTube | `youtube.com/watch`, `youtu.be/` | yt-dlp で音声DL |
| Loom | `loom.com/share/` | yt-dlp で音声DL |
| UTAGE | `utage-system.com/video/` | ページからm3u8 URL抽出 → ffmpeg |
| その他 | yt-dlp対応サイト | yt-dlp で音声DL |

### 2.2 UTAGE専用抽出ロジック

```
1. WebFetch でページHTMLを取得
2. regex で .m3u8 URL を抽出
   パターン: https://s3.ap-northeast-1.wasabisys.com/utagesystem-video/.../.../video.m3u8
3. ffmpeg -i <m3u8_url> -vn -acodec libmp3lame -q:a 4 output.mp3
```

### 2.3 音声ファイル仕様

| 項目 | 値 |
|------|-----|
| 形式 | MP3 |
| サンプルレート | 16kHz（Whisper互換） |
| チャンネル | モノラル |
| 最大サイズ | Claude API制限に準拠 |
| 保存場所 | /tmp/distill/{sessionId}/ |
| 自動削除 | 処理完了後に即時削除 |

### 2.4 長時間動画の分割処理

Claude APIの音声入力にはサイズ制限があるため、長時間動画は分割処理する。

```
60分以上の動画:
  → 10分ごとに音声を分割
  → 各チャンクを順次文字起こし
  → 結果を結合
```

## 3. 文字起こし処理

### 3.1 Claude API呼び出し

```typescript
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-20250514",
  max_tokens: 8192,
  messages: [{
    role: "user",
    content: [
      {
        type: "audio",
        source: {
          type: "base64",
          media_type: "audio/mp3",
          data: audioBase64,
        },
      },
      {
        type: "text",
        text: "この音声を日本語で正確に文字起こししてください。話者の発言をそのまま書き起こし、句読点を適切に入れてください。",
      },
    ],
  }],
});
```

### 3.2 文字起こし品質

| 項目 | 基準 |
|------|------|
| 日本語認識精度 | 95%以上 |
| 句読点挿入 | 自動 |
| 改行 | 意味的なまとまりで区切り |
| 話者区別 | 可能な場合は区別 |

## 4. コンテンツ生成

### 4.1 生成パイプライン

文字起こし完了後、1回のClaude API呼び出しで全コンテンツを生成。

```typescript
const prompt = `
以下の文字起こしテキストから、4種類のコンテンツを生成してください。

## 入力テキスト
${transcript}

## 出力形式（JSON）
{
  "summary": {
    "oneliner": "3行以内の要約",
    "detailed": "300〜500字の詳細要約",
    "chapters": [
      { "title": "章タイトル", "summary": "章の要約", "timestamp": "00:00" }
    ]
  },
  "blog": {
    "title": "ブログ記事タイトル",
    "content": "Markdown形式のブログ記事"
  },
  "sns": {
    "twitter": ["投稿1", "投稿2", "投稿3"],
    "instagram": "Instagramキャプション",
    "linkedin": "LinkedIn投稿"
  }
}
`;
```

### 4.2 要約仕様

| 形式 | 文字数 | 用途 |
|------|--------|------|
| 3行サマリー | 〜150字 | クイック確認 |
| 詳細要約 | 300〜500字 | 内容把握 |
| 章立て要約 | 各章100〜200字 | 構造理解 |

### 4.3 ブログ変換仕様

| 項目 | 仕様 |
|------|------|
| 形式 | Markdown |
| 構成 | タイトル → 導入 → 本文（見出し付き）→ まとめ |
| 文体 | です・ます調 |
| 文字数 | 1,500〜3,000字 |

### 4.4 SNS変換仕様

| SNS | 文字数 | 特徴 |
|-----|--------|------|
| X (Twitter) | 〜280字 × 3〜5投稿 | ハッシュタグ付き |
| Instagram | 〜2,200字 | 絵文字・ハッシュタグ |
| LinkedIn | 〜1,300字 | ビジネス向け |

## 6. API設計

### 6.1 POST /api/distill

メインの処理エンドポイント。SSE（Server-Sent Events）でリアルタイム進捗を返す。

**Request:**
```json
{
  "url": "https://www.youtube.com/watch?v=xxxxx"
}
```

**SSE Response Stream:**
```
event: progress
data: {"step": "extracting", "progress": 10, "message": "音声を抽出中..."}

event: progress
data: {"step": "transcribing", "progress": 40, "message": "文字起こし中..."}

event: progress
data: {"step": "generating", "progress": 70, "message": "コンテンツ生成中..."}

event: complete
data: {
  "meta": {
    "title": "動画タイトル",
    "channel": "チャンネル名",
    "duration": "15:32",
    "platform": "youtube",
    "thumbnail": "https://..."
  },
  "transcript": "全文テキスト...",
  "summary": { ... },
  "blog": { ... },
  "sns": { ... }
}
```

### 6.2 POST /api/pdf

PDF生成エンドポイント。

**Request:**
```json
{
  "meta": { "title": "...", "channel": "...", "duration": "..." },
  "transcript": "...",
  "summary": { ... },
  "blog": { ... },
  "sns": { ... }
}
```

**Response:** PDF バイナリ（Content-Type: application/pdf）

## 7. 状態管理

### 7.1 アプリケーション状態

```typescript
type AppState =
  | { status: "idle" }
  | { status: "processing"; step: ProcessStep; progress: number }
  | { status: "complete"; result: DistillResult }
  | { status: "error"; message: string };

type ProcessStep =
  | "extracting"    // 音声抽出中
  | "transcribing"  // 文字起こし中
  | "generating";   // コンテンツ生成中

type ActiveTab = "transcript" | "summary" | "blog" | "sns";
```

### 7.2 状態遷移

```
idle → processing(extracting) → processing(transcribing)
     → processing(generating) → complete
     → error（任意の段階で発生可能）
```

## 8. エラーハンドリング

| エラー | 原因 | ユーザー表示 |
|--------|------|------------|
| INVALID_URL | URL形式が不正 | 有効なURLを入力してください |
| UNSUPPORTED_PLATFORM | 非対応プラットフォーム | このURLには対応していません |
| EXTRACTION_FAILED | 音声抽出失敗 | 動画の音声を取得できませんでした |
| TRANSCRIPTION_FAILED | 文字起こし失敗 | 文字起こしに失敗しました |
| API_KEY_MISSING | APIキー未設定 | APIキーを設定してください |
| RATE_LIMIT | API制限 | しばらく待ってから再試行してください |
| AUDIO_TOO_LONG | 音声が長すぎる | 動画が長すぎます（最大3時間） |
