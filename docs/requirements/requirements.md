# Distill - 要件定義書

## 概要

**Distill** は、動画コンテンツを「蒸留」し、テキストベースのアセットに変換する個人用Webアプリケーションです。
競合・他社の動画（YouTube / Loom / UTAGE等）のURLを入力するだけで、全文文字起こし・要約・ブログ記事・SNS投稿を自動生成し、PDFで保存できます。

## プロダクト情報

| 項目 | 値 |
|------|-----|
| アプリ名 | Distill |
| タグライン | 動画を蒸留して、本質を抽出する |
| ユーザー | 個人利用（オーナー本人） |
| 用途 | 競合動画リサーチ・コンテンツ再利用 |
| 言語 | 日本語メイン |

## 機能要件

### FR-001: URL入力による動画取得

- **優先度**: Critical
- **説明**: ユーザーがURLを入力すると、動画の音声を自動抽出する
- **対応プラットフォーム**:
  - YouTube（yt-dlp）
  - Loom（yt-dlp）
  - UTAGE（HLSストリーム .m3u8 → ffmpeg）
  - その他 yt-dlp 対応サイト
- **受入基準**:
  - [ ] URLを入力フィールドに貼り付けできる
  - [ ] YouTube URLから音声を抽出できる
  - [ ] Loom URLから音声を抽出できる
  - [ ] UTAGE URLからHLSストリームの音声を抽出できる
  - [ ] 非対応URLの場合、エラーメッセージを表示する
  - [ ] 処理中にプログレス表示がある

### FR-002: 音声文字起こし（Claude API）

- **優先度**: Critical
- **説明**: 抽出した音声をClaude APIに送信し、日本語の文字起こしを生成する
- **詳細**:
  - Claude APIの音声入力機能を使用
  - 日本語音声を高精度にテキスト化
  - タイムスタンプ付き文字起こし（可能な場合）
- **受入基準**:
  - [ ] 音声ファイルからテキストが生成される
  - [ ] 日本語の精度が実用レベルである
  - [ ] 長時間動画（60分以上）にも対応する

### FR-003: 全文表示

- **優先度**: Critical
- **説明**: 文字起こし結果の全文をそのまま表示する
- **受入基準**:
  - [ ] 全文テキストがタブで閲覧できる
  - [ ] テキストのコピーができる
  - [ ] 長文でもスムーズにスクロールできる

### FR-004: 要約生成

- **優先度**: High
- **説明**: 文字起こしテキストをClaude APIで要約する
- **要約形式**:
  - 3行サマリー（ワンライナー）
  - 詳細要約（300〜500字）
  - 章立て要約（セクション分割 + 各セクション要約）
- **受入基準**:
  - [ ] 3種類の要約形式を切り替えて表示できる
  - [ ] 要約の精度が実用レベルである
  - [ ] 動画のメタ情報（タイトル、チャンネル名等）が表示される

### FR-005: ブログ記事変換

- **優先度**: High
- **説明**: 文字起こしテキストからブログ記事形式のMarkdownを生成する
- **詳細**:
  - 見出し・段落・箇条書きで構造化
  - SEOを意識したタイトル候補
  - 導入・本文・まとめの構成
- **受入基準**:
  - [ ] Markdown形式でブログ記事が生成される
  - [ ] コピー・ダウンロードができる

### FR-006: SNS投稿変換

- **優先度**: Medium
- **説明**: 文字起こしテキストからSNS投稿用テキストを生成する
- **対応SNS**:
  - X（Twitter）: 140〜280文字 × 複数投稿
  - Instagram: キャプション形式
  - LinkedIn: ビジネス向け長文
- **受入基準**:
  - [ ] SNSプラットフォーム別に最適化されたテキストが生成される
  - [ ] 複数パターンの投稿案が提示される

### FR-007: PDF出力

- **優先度**: Critical
- **説明**: 全文・要約・ブログ・SNSの各コンテンツをPDFとして保存できる
- **詳細**:
  - スタイリッシュなPDFデザイン
  - 動画メタ情報をヘッダーに含む
  - 日本語フォント対応
- **受入基準**:
  - [ ] 各タブの内容をPDFでダウンロードできる
  - [ ] 全コンテンツを1つのPDFにまとめて出力できる
  - [ ] PDFのフォーマットが崩れない

### FR-008: タブ切替UI

- **優先度**: High
- **説明**: 全文・要約・ブログ・SNSをタブで切り替えて表示する
- **タブ構成**:
  1. 全文（Transcript）
  2. 要約（Summary）
  3. ブログ（Blog）
  4. SNS（Social）
- **受入基準**:
  - [ ] タブ切替がスムーズに動作する
  - [ ] 各タブの内容が正しく表示される
  - [ ] アクティブタブが視覚的にわかる

## 非機能要件

### NFR-001: パフォーマンス

| 項目 | 目標値 |
|------|--------|
| URL解析〜音声抽出 | 30秒以内（10分動画） |
| 文字起こし完了 | 2分以内（10分動画） |
| 要約・変換生成 | 30秒以内 |
| PDF生成 | 5秒以内 |
| Lighthouse Performance | 90+ |

- APIキーはサーバーサイドで管理（クライアントに露出しない）
- 環境変数で管理（.env.local）
- 音声ファイルは処理後に削除

### NFR-003: ユーザビリティ

- シングルページで完結
- レスポンシブ対応（PC優先、モバイル可）
- ダークモード対応
- 日本語UI

### NFR-004: 可用性

- ローカル開発環境で動作（個人用）
- 将来的にデプロイ可能な構成

## ユーザーストーリー

### US-001: 競合動画のリサーチ

> 円香として、競合の動画URLを貼り付けて全文文字起こしを取得したい。
> 動画を観る時間を節約し、テキストで効率的にインプットするため。

### US-002: コンテンツの要約

> 円香として、長い動画の要約を3行・詳細・章立てで確認したい。
> 動画の要点を素早く把握するため。

### US-003: ブログ記事への転用

> 円香として、動画の内容をブログ記事形式に変換したい。
> インプットした知見を自分のコンテンツとして発信するため。

### US-004: SNS投稿の作成

> 円香として、動画の内容をSNS投稿用テキストに変換したい。
> 効率的にSNS発信するため。

### US-005: 結果の保存

> 円香として、分析結果をPDFで保存したい。
> 後から参照・共有できるようにするため。

## UTAGE対応 技術詳細

### UTAGEの動画ホスティング構造

UTAGEはWasabi S3にHLSストリーム(.m3u8)として動画をホスト。

```
UTAGEページURL
  → JavaScript config から video.m3u8 URL を抽出
  → ffmpeg で .m3u8 → 音声ファイル変換
  → Claude APIで文字起こし
```

### 抽出フロー

```javascript
// 1. UTAGEページからm3u8 URLを抽出
const pageHtml = await fetch(utageUrl);
const m3u8Match = pageHtml.match(/https:\/\/.*\.m3u8/);

// 2. ffmpegでHLS → 音声変換
// ffmpeg -i <m3u8_url> -vn -acodec libmp3lame output.mp3

// 3. Claude APIで文字起こし
```
