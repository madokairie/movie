# Distill - 要件定義書

## 概要

**Distill** は、動画コンテンツを「蒸留」し、実践可能なナレッジ資産に変換する個人用Webアプリケーションです。
競合・他社の動画（YouTube / Loom / UTAGE等）のURLを入力するだけで、全文文字起こし・ノウハウ抽出・ブログ記事・SNS投稿を自動生成し、PDF/テキストで保存できます。

## プロダクト情報

| 項目 | 値 |
|------|-----|
| アプリ名 | Distill |
| タグライン | 動画から、要点だけを取り出す |
| サブタグライン | URLを貼るだけで文字起こし・要約・ブログ・SNS投稿を自動生成 |
| ユーザー | 個人利用（オーナー本人） |
| 用途 | 競合動画リサーチ・ナレッジ抽出・コンテンツ再利用 |
| 言語 | 日本語メイン |

## 機能要件

### FR-001: URL入力による動画取得

- **優先度**: Critical
- **説明**: ユーザーがURLを入力すると、動画の音声を自動抽出する
- **対応プラットフォーム**:
  - YouTube（yt-dlp）
  - Loom（yt-dlp）
  - UTAGE（HLSストリーム .m3u8 → ffmpeg）
  - その他 yt-dlp 対応サイト（generic扱い）
- **受入基準**:
  - [x] URLを入力フィールドに貼り付けできる
  - [x] YouTube URLから音声を抽出できる
  - [x] Loom URLから音声を抽出できる
  - [x] UTAGE URLからHLSストリームの音声を抽出できる
  - [x] 非対応URLの場合、エラーメッセージを表示する
  - [x] 処理中にリアルタイムプログレスバーが表示される（SSE）
  - [x] ページ読み込み時に入力フィールドにオートフォーカスする

### FR-002: 音声文字起こし（Whisper）

- **優先度**: Critical
- **説明**: 抽出した音声をローカルのWhisper（openai-whisper）で文字起こしする
- **詳細**:
  - ローカルインストールされた Whisper CLI を使用
  - モデル: `base`、言語: `ja`（日本語）
  - 出力形式: テキスト（.txt）
  - タイムアウト: 30分
- **受入基準**:
  - [x] 音声ファイルからテキストが生成される
  - [x] 日本語の精度が実用レベルである
  - [x] 長時間動画にも対応する（30分タイムアウト）

### FR-003: 全文表示

- **優先度**: Critical
- **説明**: 文字起こし結果の全文をそのまま表示する
- **受入基準**:
  - [x] 全文テキストが「全文」タブで閲覧できる
  - [x] テキストのコピーができる（コピーボタン付き）
  - [x] 長文でもスムーズにスクロールできる

### FR-004: ナレッジ抽出・要約生成

- **優先度**: High
- **説明**: 文字起こしテキストをClaude API（claude-sonnet-4-20250514）で分析し、実践可能なナレッジを抽出する
- **要約形式**:
  - 核心ノウハウ（ワンライナー 80字以内）
  - 詳細ノウハウ（1500〜3000字、マークダウン構造化）
    - 核心ノウハウ、テクニック詳細、実践ステップ、注意点
  - 章別テクニック（各300〜500字、タイムスタンプ付き）
- **受入基準**:
  - [x] 3種類の要約形式（ワンライナー・詳細・章立て）を1画面で表示
  - [x] 「すぐに実践できるレベル」の具体性がある
  - [x] 動画のメタ情報（タイトル、チャンネル名、再生時間、プラットフォーム）が表示される
  - [x] カテゴリ自動分類（マーケティング / セールス / コピーライティング等）

### FR-005: ブログ記事変換

- **優先度**: High
- **説明**: 文字起こしテキストからナレッジブログ記事形式のMarkdownを生成する
- **詳細**:
  - 構成: リード → 核心 → 実践テクニック → 事例・データ → チェックリスト
  - 2000〜4000字、です・ます調
  - Markdown形式（見出し・段落・箇条書きで構造化）
- **受入基準**:
  - [x] Markdown形式でブログ記事が生成される
  - [x] 「ブログ」タブで表示される
  - [x] コピーができる

### FR-006: SNS投稿変換

- **優先度**: Medium
- **説明**: 文字起こしテキストからSNS投稿用テキストを生成する
- **対応SNS**:
  - X（Twitter）: 6パターン生成
    1. フック型（140字以内）
    2. リスト型（140字以内）
    3. 逆説型（140字以内）
    4. ストーリー型（140字以内）
    5. 問いかけ型（140字以内）
    6. 長文X記事（1500〜2500字）
  - Instagram: キャプション形式（2200字以内）
  - LinkedIn: ビジネス向け（1300字以内）
- **制約**: ハッシュタグ（#記号）は全SNS投稿で禁止
- **受入基準**:
  - [x] SNSプラットフォーム別に最適化されたテキストが生成される
  - [x] X投稿は6パターン（短文5 + 長文1）が提示される
  - [x] 「SNS」タブで表示される

### FR-007: エクスポート（PDF / テキスト）

- **優先度**: Critical
- **説明**: 全コンテンツをPDFまたはテキストファイルとして保存できる
- **実装方式**:
  - **PDF出力**: `window.open()` で印刷用HTML生成 → ブラウザの印刷ダイアログ（PDF保存）
  - **テキスト出力**: Blob生成 → ダウンロード（.txt）
- **PDF内容**:
  - 動画タイトル、メタ情報
  - 核心ノウハウ、詳細ノウハウ、章別テクニック
  - ブログ記事、全文書き起こし
  - 日本語フォント対応（Hiragino Sans / Noto Sans JP）
- **テキスト内容**:
  - 全セクション（要約・全文・ブログ・SNS）を統合した「ナレッジノート」形式
- **受入基準**:
  - [x] ドロップダウンメニューで「テキストで保存」「PDFで保存」を選択できる
  - [x] PDFが印刷ダイアログから保存できる
  - [x] テキストファイルが自動ダウンロードされる
  - [x] ファイル名に動画タイトルが含まれる

### FR-008: タブ切替UI

- **優先度**: High
- **説明**: 全文・要約・ブログ・SNSをタブで切り替えて表示する
- **タブ構成**:
  1. 全文（Transcript）- FileTextアイコン
  2. 要約（Summary）- ListTreeアイコン
  3. ブログ（Blog）- BookOpenアイコン
  4. SNS（Social）- Share2アイコン
- **タブスタイル**: ライン型（下線ボーダーでアクティブ表示、銅色アクセント）
- **受入基準**:
  - [x] タブ切替がスムーズに動作する
  - [x] アクティブタブが銅色の下線で視覚的にわかる
  - [x] モバイルでタブが横スクロール可能
  - [x] エクスポートボタンがタブバー右端に配置

### FR-009: 履歴・ライブラリ

- **優先度**: High
- **説明**: 過去の分析結果をライブラリとして管理・閲覧できる
- **詳細**:
  - アイドル状態（結果未表示時）にライブラリ一覧を表示
  - カード形式で最大50件表示（2カラムグリッド）
  - カテゴリフィルタリング（タグボタンで絞り込み）
  - プラットフォーム別カラーバッジ（YouTube: 赤、Loom: 紫、UTAGE: 銅）
  - 相対時間表示（「3分前」「昨日」「2週間前」等）
  - カードクリックで結果を再表示（ローディングスピナー付き）
- **受入基準**:
  - [x] ライブラリ一覧が件数バッジ付きヘッダーで表示される
  - [x] カテゴリタグで絞り込みができる
  - [x] カードクリックで過去の分析結果が表示される
  - [x] 履歴が0件の場合はタグラインを表示する

### FR-010: タイトル編集

- **優先度**: Medium
- **説明**: 分析結果のタイトルをインラインで編集できる
- **詳細**:
  - タイトルホバー時に鉛筆アイコン表示
  - クリックで編集モード（インプットフィールド）
  - Enter / blur で保存、Escape でキャンセル
  - サーバーサイドの履歴データも更新（PATCH API）
- **受入基準**:
  - [x] タイトルをクリックして編集できる
  - [x] 保存後にUIとサーバーデータの両方が更新される

### FR-011: 履歴削除

- **優先度**: Medium
- **説明**: ライブラリのカードからエントリを削除できる
- **詳細**:
  - カードホバー時に右上にゴミ箱アイコン表示
  - クリックでサーバーから削除（DELETE API）
  - UIから即座に削除（楽観的更新）
- **受入基準**:
  - [x] ゴミ箱アイコンクリックでエントリが削除される
  - [x] 削除がサーバーとUIの両方に反映される

### FR-012: エラーハンドリング・リトライ

- **優先度**: High
- **説明**: 処理エラー時にエラーメッセージとリトライ機能を提供する
- **詳細**:
  - エラー表示: 赤背景のアラートバーにエラーメッセージ
  - URLが入力済み → 「再試行」ボタン表示
  - URLが空 → 「戻る」ボタン表示
  - ヘッダーの「Distill」ロゴクリックでいつでもホームに戻れる
- **受入基準**:
  - [x] エラーメッセージが表示される
  - [x] 再試行ボタンで同じURLを再処理できる
  - [x] ロゴクリックでアイドル状態にリセットできる

## 非機能要件

### NFR-001: パフォーマンス

| 項目 | 目標値 |
|------|--------|
| URL解析〜音声抽出 | 30秒以内（10分動画） |
| 文字起こし完了（Whisper） | 2〜5分（10分動画、baseモデル） |
| 要約・変換生成（Claude API） | 30秒以内 |
| エクスポート生成 | 即時 |
| Lighthouse Performance | 90+ |
| API Route最大実行時間 | 600秒（10分） |

### NFR-002: セキュリティ

- Anthropic APIキーはサーバーサイドで管理（`.env.local`）
- クライアントにAPIキーを露出しない
- 音声ファイルは処理後に自動削除（cleanup関数）
- URLバリデーション（`https?://` パターン）

### NFR-003: ユーザビリティ

- シングルページで完結（SPA）
- レスポンシブ対応（PC優先、モバイル可）
- **ダークモード固定**（トグルなし、`className="dark"` ハードコード）
- 日本語UI
- 結果表示時のフェードインアニメーション
- リアルタイム処理進捗表示（SSE）

### NFR-004: 可用性

- ローカル開発環境で動作（個人用）
- 将来的にデプロイ可能な構成
- 履歴データはローカルJSONファイルで永続化（`data/history.json`）

## ユーザーストーリー

### US-001: 競合動画のリサーチ

> 円香として、競合の動画URLを貼り付けて全文文字起こしを取得したい。
> 動画を観る時間を節約し、テキストで効率的にインプットするため。

### US-002: ナレッジの抽出

> 円香として、長い動画から実践可能なノウハウを抽出したい。
> 動画の要点を「読んだだけで実践できるレベル」で把握するため。

### US-003: ブログ記事への転用

> 円香として、動画の内容をナレッジブログ記事形式に変換したい。
> インプットした知見を自分のコンテンツとして発信するため。

### US-004: SNS投稿の作成

> 円香として、動画の内容を複数パターンのSNS投稿用テキストに変換したい。
> X・Instagram・LinkedInで効率的に発信するため。

### US-005: 結果の保存

> 円香として、分析結果をPDFまたはテキストで保存したい。
> 後から参照・活用できるようにするため。

### US-006: 過去の分析の管理

> 円香として、過去の分析結果をライブラリで管理・再閲覧したい。
> 蓄積したナレッジを効率的に参照するため。

### 処理フロー（ユーザー視点）

```
1. URLを貼り付け
2. 「Distill」ボタンをクリック
3. プログレスバーで進捗確認（SSEリアルタイム）
   - 音声を抽出中...
   - 文字起こし中...
   - コンテンツを生成中...
4. 結果表示（フェードインアニメーション）
5. タブ切替で各コンテンツ確認
6. PDF/テキスト保存（任意）
7. ライブラリに自動保存
```

### アプリケーション状態遷移

```
idle ──(URL入力+Distill)──→ processing
processing ──(完了)──→ complete
processing ──(エラー)──→ error
error ──(再試行)──→ processing
error ──(戻る)──→ idle
complete ──(新しく分析 / ロゴクリック)──→ idle
idle ──(ライブラリからカード選択)──→ complete
```

### API Routes設計

| Endpoint | Method | 説明 |
|----------|--------|------|
| `/api/distill` | POST | URL受取→音声抽出→文字起こし→変換（SSEストリーム） |
| `/api/history` | GET | 履歴一覧取得（サマリー） |
| `/api/history?id=xxx` | GET | 個別履歴取得（全データ） |
| `/api/history` | PATCH | タイトル更新 `{ id, title }` |
| `/api/history?id=xxx` | DELETE | 履歴エントリ削除 |

## UTAGE対応 技術詳細

### UTAGEの動画ホスティング構造

UTAGEはWasabi S3にHLSストリーム(.m3u8)として動画をホスト。

```
UTAGEページURL
  → JavaScript config から video.m3u8 URL を抽出
  → ffmpeg で .m3u8 → 音声ファイル変換
  → Whisperで文字起こし
```

### 抽出フロー

```javascript
// 1. UTAGEページからm3u8 URLを抽出
const pageHtml = await fetch(utageUrl);
const m3u8Match = pageHtml.match(/https:\/\/.*\.m3u8/);

// 2. ffmpegでHLS → 音声変換
// ffmpeg -i <m3u8_url> -vn -acodec libmp3lame output.mp3

// 3. Whisperで文字起こし
// whisper output.mp3 --language ja --model base --output_format txt
```
